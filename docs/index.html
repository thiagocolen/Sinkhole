Your Data, Your Bridge.

A secure, manifest-based synchronization engine connecting Android's local storage with Google Drive. Zero friction, total ownership.

Repository Getting Started

Why Sinkhole?

Sinkhole was engineered to solve the "last mile" problem of mobile data synchronization. It leverages native Android security primitives to ensure your files remain yours.

SAF Integration

Uses the Storage Access Framework (SAF) to request granular folder permissions. No full disk access required.

Manifest Tracking

Intelligent change detection using a local .sync-manifest.json. Syncs only what changed, saving battery and data.

OAuth 2.0

Secure authentication directly with Google. Your credentials never touch our code; we only use temporary access tokens.

Technical Internals

Sinkhole operates on a Last Write Wins (LWW) conflict resolution strategy. Every sync operation follows a rigorous verification pipeline.

Manifest Schema

The manifest tracks both local and remote states to detect modifications, deletions, and collisions without re-downloading content.

{
  "files": {
    "notes/ideas.md": {
      "id": "drive_file_id_123",
      "localMtime": 1711396800000,
      "remoteMtime": 1711396810000,
      "size": 1024,
      "hash": "d41d8cd98f00b204e9800998ecf8427e"
    }
  }
}

Intelligent Ignoring

Supports .sinkhole-ignore files with glob pattern matching, similar to .gitignore. System files like .DS_Store are ignored by default.

// Example ignore logic in TypeScript
const isIgnored = (name, isDir, relativePath, customPatterns) => {
  if (SYSTEM_IGNORE_LIST.includes(name)) return true;
  return customPatterns.some(pattern => matchGlob(name, pattern));
};

Getting Started

  • 1

    Environment Prep

    Enable Unknown Sources in your Android settings and install the latest APK from the releases page.

  • 2

    Vault Selection

    Select your local directory (e.g., your Obsidian Vault). Sinkhole will generate the initial manifest.

  • 3

    Cloud Pairing

    Authorize the app via the Google Sign-In prompt. A dedicated SinkholeFolder will be created in your Drive.

  • 4

    Continuous Sync

    Run the sync manually or check the logs for detailed transfer metrics. Your data is now bridged!
